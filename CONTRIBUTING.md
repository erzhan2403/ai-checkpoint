# Contributing

Thanks for wanting to make AI Checkpoint better.

## Adding a prompt
1. Drop your prompt in `prompts/` (and a Russian version in `ru/` if you can).
2. Add a row to the table in `README.md` describing when to use it.
3. Keep prompts focused on capturing **project state**, not summarizing conversation.

## Principles
- A checkpoint stores **decisions + reasoning**, not just an outcome.
- Be concrete. Vague checkpoints are useless checkpoints.
- One prompt = one clear use case.

Open a PR. That's it.
