# Checkpoint — Разработка

Вставь это в **конце** сессии по коду/автоматизации. AI выдаст структурированный снимок, который можно загрузить в новый чат.

---

```
Ты создаёшь CHECKPOINT нашей работы — структурированный снимок ТЕКУЩЕГО СОСТОЯНИЯ ПРОЕКТА, а не пересказ разговора.

Цель: человек (или другой AI) должен прочитать этот checkpoint и сразу продолжить работу, не читая историю чата.

Сделай checkpoint строго по этой структуре. Будь конкретным, факты важнее повествования.

## 1. Цель проекта
Что мы строим и зачем, в 2–3 предложениях.

## 2. Технологический стек
Языки, фреймворки, сервисы, инфраструктура, которые реально используются.

## 3. Текущая архитектура
Как части связаны между собой. Ключевые компоненты и как они общаются.

## 4. Принятые решения (и ПОЧЕМУ)
Самая важная часть. Перечисли каждое значимое решение и причину за ним, чтобы его случайно не откатили позже.

## 5. Что реализовано
Конкретные рабочие фичи / модули на текущий момент.

## 6. Изменённые / созданные файлы в этой сессии
Имена файлов или пути, с однострочным пояснением, что делает каждый.

## 7. Известные проблемы / открытые вопросы
Баги, блокеры, нерешённые вопросы.

## 8. Следующий шаг
Одна самая важная вещь, которую нужно сделать дальше, сформулированная конкретно.

## 9. Промпт для продолжения
Короткий готовый промпт, который я вставлю в НОВЫЙ чат (вместе с этим checkpoint), чтобы продолжить без потерь.

Выведи checkpoint в чистом Markdown. Ничего вне этой структуры.
```
