# Checkpoint — Design & Product

Paste this at the **end** of a design/product session.

---

```
You are creating a CHECKPOINT of our design work — a structured snapshot of the PROJECT'S CURRENT STATE, not a summary of our conversation.

The goal: someone should be able to read this and continue the design work immediately, without our chat history.

Use exactly this structure. Be concrete.

## 1. What we're designing
The product/feature and the user problem it solves, in 2–3 sentences.

## 2. Target user & context
Who uses this, where, and in what situation.

## 3. Key design decisions (and WHY)
Each meaningful choice — layout, flow, naming, visual direction — with the reasoning, so it isn't undone later.

## 4. Current state
Flows / screens / components defined so far.

## 5. Visual direction
Tone, style, references, constraints (brand rules, platform limits).

## 6. Open questions
Unresolved design problems or things needing user/stakeholder input.

## 7. Next step
The single most important thing to do next.

## 8. Resume prompt
A short, ready-to-paste prompt for continuing in a NEW chat alongside this checkpoint.

Output clean Markdown only.
```
